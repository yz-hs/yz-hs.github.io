奇怪错误

指不应该出、很奇怪的错误。

# 1

时间：2020-10-03

详情：打 trie，为了卡空间，竟用 char 类型数组作为儿子数组。链接

struct Trie片段：
    struct Node
    {
        char son[10],exist;
~~~~~~~~~^建议：使用int
    }node[500000+10];
--------------------------------------
    void insert(char s[])片段：
           for(int i=0;i<=len-1;i++)
               if(!node[root].son[sons])
                   node[root].son[sons]=++index;
~~~~~~~~~~~~~~~~~~~~~~~~~~~~^警告：有溢出风险

分析：类似“十年 OI 一场空，不开 LL 见祖宗”。即开变量不考虑数据范围。

# 3

时间：2020-11-07

详情：CSP-S 考场。由于对 T1 过于自信，花了 2.5h 做（考试时间 4h）。最终放弃，也导致错失了 AC T2 的机会。

分析：一定要合理分配时间，在考场上千万不要死磕一道题！

# 4

时间：2020-11-27

详情：做 [THUPC2019]鸭棋 这个题时，发现判断是否能下棋的函数总是返回“是”。看了看我的一长串 switch 中的 case 语句。嗯，好的，原来是…

$$\color{red}{\Huge{\texttt{没加 “break;” 语句！}}}$$

分析：注意 C++ 的毒瘤语法啊！！！

知识错误

一般是初赛考的知识点之类。

# 1：dijkstra 若未使用任何优化，时间复杂度为 $O(n^2)$。借助优先队列等，时间复杂度则为 $O((n+m)logn)$。

# 2：1948年，克劳德·香农将热力学中的熵引入信息通信领域，标志着信息论研究的开端。
